# 维护操作类型

在维护数据结构过程中，我们通常会进行一些操作，我们大体可以将这些操作分为以下几类: 

- **单点查询/修改**: 对单个位置上的值进行查询/修改
- **区间查询/修改**: 对区间进行查询/修改

# 前缀和与差分

设一个二元操作 $+$，逆操作为 $-$. 数据为 $a_0,a_1,...,a_n$.

## 前缀和

我们需要 $O(1)$ 进行如 $a_l + a_{l+1} + ... + a_r$ 这样的 **区间查询** 时，可用前缀和维护.

我们预先 $O(n)$ 维护一个数组 $S_i = a_0 + a_1 + ... + a_i$，每次查询 $[l,r]$ 区间的二元运算和时，直接计算 $S_r - S_{l-1}$ 即可.

## 差分

我们需要 $O(1)$ 对数据进行如在 $[l, r]$ 区间的每个值 $+k$ 这样的 **区间修改** 操作，最后需要 **很少** 的 **单点查询** 时，可用差分维护.

我们预先维护一个差分数组 $D_i = a_i - a_{i-1}$，每次区间修改时，进行操作 $D_l \leftarrow D_l + k, D_{r+1} \leftarrow D_{r+1} - k$. 单点查询第 $i$ 个值时 $O(n)$ 计算 $D_0+D_1+...+D_i$.

{% note %}
显然前缀和与差分为一对互逆操作
{% endnote %}

## 条件

并非所有二元运算都可满足前缀和与差分操作.

- 可满足的运算如：加法，模意义下加法，无零乘法，异或等;

- 不可满足的运算如：取最大/小值，最大公因数，最小公倍数，减法等.

结论是，一个二元运算可满足前缀和与差分操作，其充要条件为：该运算在对应集合上构成一个 **群**. 即满足：
- 结合律
- 单位元
- 逆元

## 二维前缀和

给定一个二维矩阵 $a_{i,j}$，我们需要 $O(1)$ 求左上角为 $(x_l,y_l)$, 右下角为 $(x_r,y_r)$ 的子矩阵之和时，可以使用二维前缀和维护.

类似一维前缀和，我们维护 $$S_{x,y} = \sum_{0 \le i \le x, 0 \le j \le y} a_{i,j}$$ 具体地说为 $S_{x,y} = a_{x,y} + S_{x-1,y}+S_{y-1,x}-S_{x-1,y-1}$

查询时，我们通过容斥计算 $S_{x_r,y_r} - S_{x_{l-1}, y_r} - S_{x_r, y_{l-1}}+S_{x_{l-1},y_{l-1}}$.

## 树上前缀和

给定一颗树, 我们需要很快地求出 $(u,v)$ 两节点构成的简单路径上值的和. 下面我们以 $(u,v)$ 代指 $(u,v)$ 简单路径上值的和，$fa(x)$ 表示 $x$ 的父节点.

类似一维前缀和，我们维护 $S_p = (root,p)$，每次查询求出 $lca(u,v)$，答案即 $S_u+S_v-S_{lca}-S_{fa(lca)}$.

树上差分同理.

## 高维前缀和

对于二维以上的前缀和，形如 $k(k > 2)$ 维前缀和，我们需要求 $$\sum_{l_1 \le i_1 \le r_1} \sum_{l_2 \le i_2 \le r_2} ··· \sum_{l_k \le i_k \le r_k}a_{i_1,i_2,...i_k}$$ 时, 若我们依然使用容斥法维护，复杂度会达到 $O(2^kN)$，其中 $N$ 为数组大小. 这样的复杂度通常我们无法忍受.

注意上面公式，我们发现 $k$ 维前缀和即求 $k$ 次和. 我们每次固定一维进行求和，求若干次一维前缀和即可. 复杂度为 $O(kN)$，一般可以接受.

查询依旧需要容斥，复杂度为 $O(2^k)$.


### 子集和DP(SOSDP)

当 $k$ 的值很大时，会出现一类 **子集和** (Sum Over Subsets, SOS) 问题，这是高维前缀和的特例. 

问题描述：考虑大小为 $n$ 的集合的全体子集上定义的函数 $f$，现在要求出其子集和函数 $g$，其满足 $$g(S) = \sum_{T \subseteq S}f(T)$$，即 $g(S)$ 等于其所有子集 $T \subseteq S$ 上的函数值 $f(T)$ 的和.

首先，子集和问题可以写成高维前缀和形式. 注意到，$S$ 的子集可以通过状态压缩表示为长度为 $n$ 的 0-1 字符串 $s$. 将字符串的每一位看作一个维度，则子集的包含关系等价于下标的大小关系，即 $$T \subseteq S \Leftrightarrow \forall i(t_i \le s_i)$$. 

所以对子集求和，即对这个 $n$ 维数组求高维前缀和. 时间复杂度为 $O(n2^n)$

子集和的逆操作依旧需要通过容斥进行.
